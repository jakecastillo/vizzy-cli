# vizzy v0.3.0 — exposure-safety platform (design spec)

> Date: 2026-06-17 · Status: approved (research-driven roadmap, Tiers 1–3).
> Builds on the v0.2.0 pre-visibility layer. Target version: **0.3.0**.

## Thesis

vizzy's moat is its **pre-visibility safety layer**, not the visibility toggle
(`gh repo edit` + `xargs` already flip bits). v0.3.0 makes that moat (1) **deeper**
(history + content scanning, not filename/HEAD-only), (2) **reachable & embeddable**
(scriptable/headless, `--json`/SARIF, a GitHub Action, a `gh` extension), and
(3) **decoupled** from bulk visibility (a `vizzy check` pre-publish command, org
audit) so it helps people who never need a bulk flip. Plus UX/trust polish.

**Invariants (must not regress):** the interactive TUI stays the default; the
private flow is unchanged; no GitHub writes happen on any scan/audit/check path;
every new core module is pure + unit-tested; no new *required* runtime deps (a
spawned `gitleaks`/`trufflehog` is optional and detected, never bundled); the gate
`npm run lint && typecheck && test && build` is green at every bead.

## Exit-code contract (new, documented)

`0` ok/clean · `1` danger finding or apply failure · `2` usage error · `3`
auth/network error. `--audit`, headless apply, and `vizzy check` all honor it.

## Modules & contracts

### A. Scriptable foundation + machine output

- **`core/report.ts`** (new, pure):
  `toJsonReport(assessments: RepoAssessment[]): object` and
  `toSarif(assessments: RepoAssessment[]): object` — SARIF 2.1.0 (one `run`,
  rules from finding `kind`, `result.level` = error for danger / warning for
  caution, `locations` = repo name). Stable, serializable.
- **`core/select.ts`** (new, pure):
  `resolveSelection(eligible: Repo[], spec: { repos?: string[]; allEligible?: boolean }): { selected: Repo[]; unknown: string[] }` —
  `--repos` names matched against eligible; `allEligible` → all. Unknown names
  surfaced (caller errors with exit 2). Reading `@file`/stdin into `string[]`
  happens in the CLI layer, not here.
- **`headless.ts`** (new): `runHeadless(deps, flags, opts): Promise<number>` where
  `deps = { loadRepos, setter, treeFetch }`. Resolves target + selection, runs
  `assessRepos`, then applies the **safe subset**: clean → apply; caution → apply
  only with `--yes`; **danger → SKIP and report unless `--force-public` (or
  `--allow-danger`)** — automation can never silently expose a detected-secret
  repo. Emits text or `--json`. Returns the exit code. No TTY required.
- **`cli.ts`**: add `--repos <list>`, `--all-eligible`, `--yes`, `--allow-danger`,
  `--json`, `--format <text|json|sarif>`, `--plain`, `--org <name>`, `--deep`,
  `--archive`/`--unarchive`, `--fail-on-new`, `--check [repo]`. Keep existing.
- **`bin.tsx`**: route non-interactive → `runHeadless`/`runAudit`/`runCheck`;
  only the interactive multi-select path requires a TTY. When no TTY **and** no
  headless-sufficient flags, print actionable guidance (how to run headless),
  exit 2 — instead of the bare "must be a terminal".

### B. Deeper scan (the moat)

- **`core/content.ts`** (new, pure): `scanContent(text: string): ContentHit[]` —
  high-confidence patterns only (AWS `AKIA[0-9A-Z]{16}`, GitHub `ghp_`/`github_pat_`,
  Stripe `sk_live_`, Slack `xox[baprs]-`, Google `AIza…`, private-key headers
  `-----BEGIN … PRIVATE KEY-----`). Tuned for precision (low false positives).
- **`github.ts`**: `getBlobText(octokit, owner, repo, sha): Promise<string>`
  (base64 decode); `listHistoryFilenames(octokit, owner, repo, maxCommits=100):
  Promise<{ paths: string[]; truncated: boolean }>` (walk recent commits' changed
  files via the commits API). Both read-only; errors propagate (→ scan-incomplete).
- **`core/scan.ts`** / **`core/checks.ts`**: a new `Finding.kind` value
  `secret-content` (danger) and `secret-in-history` (danger). `assess` accepts
  optional `contentHits` + `historyHits`. Content + history scanning are **opt-in**
  (`--deep`, and always-on inside `vizzy check`) to keep the default bulk path fast
  and within rate limits; injected fetchers keep it unit-testable with no network.
- **`core/scanrules.ts`** (new, pure): `loadScanRules(text)` parses `.vizzyscan`
  (custom danger globs + an allowlist of paths to never flag); `sensitive.ts`
  `classifyPath` gains an optional `extra` param (custom rules + allowlist). Allow
  beats deny. Mirrors `.vizzyignore` ergonomics.

### C. New surfaces

- **`check.ts`** (new) + `vizzy check [repo]`: pre-publish readiness for ONE repo
  (default: infer from the cwd git remote; or `--check owner/repo`). Runs the full
  scan (tree + content + history), checks LICENSE present, README/CONTRIBUTING/
  CODE_OF_CONDUCT present, and large files (blob `size` from the tree, flag >50MB).
  Prints a checklist, exits per the contract. Decoupled from bulk visibility.
- **`github.ts`** `listOrgRepos(octokit, org)` + `--org` for the **read-only audit
  path only** (`vizzy --audit --org acme`). The write paths stay personal-only.
- **`core/snapshot.ts`** (new, pure) + drift: read/write `.vizzy/state.json`
  (`{repo → {visibility, findingFingerprints}}`); `diffSnapshot(prev, curr)` →
  deltas. `--audit` records + diffs; `--fail-on-new` exits non-zero only on new
  exposure/findings vs. the snapshot.
- **Bulk archive**: a second operation. `types.ts` `Operation = 'visibility' |
  'archive'`; `github.ts` `setArchived(octokit, owner, repo, archived)`; `apply.ts`
  generalized to apply any per-repo mutation. `--archive`/`--unarchive` select the
  op (reuses selection + the apply progress UI; no exposure scan needed — archiving
  is reversible and non-exposing).

### D. UX / trust polish

- **`core/consequences.ts`** (new, pure): `consequencesFor(repo, target): string[]`
  — concrete effects using data vizzy already has (going private: "erases N stars,
  detaches N forks, unpublishes Pages"; going public: "publishes Actions logs,
  disables push rulesets"). Rendered in the confirm screen.
- **NO_COLOR + `--plain`**: a `core/theme.ts` (or `ui/theme.ts`) helper reads
  `NO_COLOR`/`flags.plain` → color on/off; UI components consult it; `--plain`
  also disables the spinner. (Numbered-list selection fallback is a stretch goal;
  minimum bar: colors/spinner off + NO_COLOR honored, output still usable.)
- **Docs**: README "Trust & safety" (MIT, token read-only from `gh`, never logged/
  persisted, no telemetry, `--dry-run`, `--audit` read-only) + least-privilege PAT
  guidance (fine-grained: Administration R/W, scoped) + an install matrix.

### E. Distribution / infra (in-repo artifacts; remote steps gated)

- **gh extension**: `gh-extension/gh-vizzy` shim + README "Install as a gh
  extension" section. (Publishing the separate `gh-vizzy` repo is a remote/gated
  step left for a human.)
- **GitHub Action**: `action.yml` (composite, wraps `vizzy --audit --format sarif`)
  + `examples/exposure-audit.yml` workflow + docs. (Marketplace listing is gated.)
- **npm provenance**: `.github/workflows/release.yml` (OIDC Trusted Publishing,
  `--provenance`) — **staged, never run**; publishing stays a gated human action.
- **Demo**: `demo/vizzy.tape` (VHS) + `scripts/render-demo.sh` + a README demo slot.
  (Rendering the binary GIF needs `vhs`; left as a documented manual/CI step.)

## Testing

Vitest per module; ink-testing-library for UI. Headless/check/audit paths tested
with injected loaders/fetchers (no network). SARIF output validated structurally.
The exit-code contract is asserted. Gate green at every bead.

## Bead breakdown (epic + ~18, dependency-ordered)

Tier A: (1) `core/report.ts` JSON+SARIF · (2) `--format` wired into audit + exit
codes · (3) `core/select.ts` · (4) `headless.ts` + non-TTY fallback.
Tier B: (5) `core/content.ts` · (6) `getBlobText` + content scan wiring · (7)
history scan · (8) `.vizzyscan` rules.
Tier C: (9) `vizzy check` · (10) `--org` audit · (11) drift/snapshot · (12) bulk
archive.
Tier D: (13) consequences in confirm · (14) NO_COLOR/`--plain` · (15) trust+PAT docs.
Tier E: (16) gh extension + GitHub Action · (17) release workflow + demo `.tape`.
Tier F: (18) README/CHANGELOG/0.3.0 bump + whole-epic review.

## Rollout / safety

All on a shift branch; `main` via human sign-off. No publish, no tag, no remote
repo creation — those are gated. v0.3.0 staged in-repo only.
